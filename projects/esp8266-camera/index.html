Project 01

ESP8266 Camera Streaming & Object Tracking

A compact surveillance and tracking system built around ESP8266 and ArduCAM OV2640. The key point: motion detection and target tracking are calculated directly on the microcontroller, while the browser only renders the result.

Open GitHub

What it proves

This project proves that useful computer-vision logic can run on very constrained hardware when the algorithm is shaped around the device. ESP8266 does not have room for neural networks or heavy classical CV pipelines, so the system uses grayscale frame differencing, bounding-box extraction, and ROI-based tracking.

The camera can operate in a regular JPEG streaming mode, but detection and tracking switch it into YUV422 capture. The firmware extracts only the Y channel, compares the current and previous frames, filters noise and lighting changes, then sends a compact binary response to the web interface.

Operating modes

JPEG stream

Multipart camera stream with selectable 160x120, 320x240, and 640x480 resolutions.

Motion detection

ESP8266 scans the grayscale frame, detects changed pixels, and returns a bounding box around movement.

Target tracking

The user clicks a target in the browser; ESP8266 tracks only a local ROI around that object.

Algorithm design

The firmware captures a 160x120 YUV422 frame and keeps only brightness values, so each analysis frame is 19,200 bytes. Two static frame buffers avoid heap fragmentation during continuous operation. Motion is detected by comparing brightness differences against a threshold, then collecting min/max coordinates into a bounding box.

To reduce false positives, the algorithm ignores small movement below a pixel count threshold and also rejects global lighting changes when too much of the frame changes at once. Tracking mode narrows the scan to a padded region around the previous target position, which lowers CPU load and keeps the box from jumping to unrelated background motion.

Data protocol

Instead of sending slow JSON for every frame, the ESP returns a custom binary packet: an 8-byte header with motion state, changed-pixel count, bounding-box validity and coordinates, followed by 19,200 bytes of grayscale pixels. The browser decodes the packet with Uint8Array and draws the frame and boxes on an HTML5 Canvas.

Core responsibilities

  • Implemented ESP8266 SoftAP web server with stream, mode, detection, and target endpoints.
  • Integrated ArduCAM OV2640 over SPI/I2C and switched between JPEG and YUV capture paths.
  • Built motion detection using frame differencing, thresholds, bounding boxes, and lighting-change filtering.
  • Implemented ROI tracking driven by user clicks from the browser Canvas interface.
  • Packed detection results and grayscale pixels into a compact binary response for faster rendering.
